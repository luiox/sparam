# sparam 总设计文档

一款简单的MCU参数监测和标定工具，通过串口和ELF文件实现通用变量读写，无需专用调试器。

## 1. 概述

### 1.1 设计目标

- **零代码侵入**：用户无需为调参编写专门的通信代码，上位机直接解析ELF/MAP文件获取变量地址
- **传输无关**：协议与传输介质解耦，仅需可靠的数据传输通道
- **多设备支持**：支持多下位机并行监测，设备ID范围0-255
- **双界面**：提供GUI和CLI两种使用方式，支持脚本化自动化

### 1.2 系统架构

```
┌─────────────────┐      串口/UART      ┌─────────────────┐
│    上位机工具    │ ◄────────────────► │    下位机固件    │
│  (Python GUI/CLI)│                     │   (C源码形式)    │
└────────┬────────┘                     └────────┬────────┘
         │                                       │
    解析ELF/MAP                             STM32 HAL层
         │                                       │
    ┌────┴────┐                             ┌────┴────┐
    │变量选择  │                             │DMA+中断 │
    │参数配置  │                             │定时器   │
    │实时显示  │                             │         │
    └──────────┘                             └─────────┘
```

## 2. 通信协议

### 2.1 帧格式

所有多字节字段均为**小端序**。

```
| 帧头 (2B) | 长度 (1B) | 设备ID (1B) | 命令 (1B) | 数据 (N B) | CRC16 (2B) |
|  AA  55   |           |             |           |            |   MODBUS   |
```

| 字段 | 大小 | 说明 |
|------|------|------|
| 帧头 | 2B | 固定 `0xAA 0x55` |
| 长度 | 1B | 命令 + 数据 + CRC 的总长度（不含帧头和长度字段） |
| 设备ID | 1B | 目标设备地址，0-255 |
| 命令 | 1B | 操作码 |
| 数据 | NB | 命令相关数据 |
| CRC16 | 2B | CRC16-MODBUS校验，从设备ID开始计算 |

### 2.2 命令编码

命令字节分为高4位（功能码）和低4位（参数）：

| 功能码 | 含义 |
|--------|------|
| 0x0x | 系统命令 |
| 0x1x | 读操作 |
| 0x2x | 写操作 |
| 0xAx | 应答 |

#### 2.2.1 系统命令 (0x0x)

| 命令 | 值 | 用途 | 数据格式 |
|------|-----|------|----------|
| 心跳 | 0x00 | 检测设备是否在线 | 空 |
| 查询信息 | 0x01 | 查询设备信息/协议版本 | 空 |

**查询信息返回**：
```
| 协议版本 (1B) | 设备名称长度 (1B) | 设备名称 (NB) |
```

#### 2.2.2 读操作 (0x1x)

低4位表示采样速率：

| 低4位 | 采样间隔 | 说明 |
|-------|----------|------|
| 0x0 | 单次 | 不启动周期，立即返回一次 |
| 0x1 | 1ms | 周期性上传 |
| 0x2 | 5ms | 周期性上传 |
| 0x3 | 10ms | 周期性上传 |
| 0x4 | 20ms | 周期性上传 |
| 0x5 | 50ms | 周期性上传 |
| 0x6 | 100ms | 周期性上传 |
| 0x7 | 200ms | 周期性上传 |
| 0x8 | 500ms | 周期性上传 |

| 命令 | 值 | 用途 | 数据格式 |
|------|-----|------|----------|
| 读单次 | 0x10 | 读取一次立即返回 | 请求数据见下表 |
| 周期读 | 0x11-0x18 | 按指定速率周期上传 | 请求数据见下表 |
| 停止采样 | 0x1F | 停止所有周期性上传 | 空 |

**读请求数据格式**：
```
| 地址1 (4B) | 地址2 (4B) | ... | 地址N (4B) |
```
地址为32位小端序。

**读返回数据格式**：
```
| 地址1 (4B) | 数据1 (NB) | 地址2 (4B) | 数据2 (NB) | ... |
```
数据长度由变量类型决定，上位机已知类型。

#### 2.2.3 写操作 (0x2x)

| 命令 | 值 | 用途 | 数据格式 |
|------|-----|------|----------|
| 写单变量 | 0x20 | 写入一个变量 | 见下表 |
| 批量写 | 0x21 | 写入多个变量 | 见下表 |

**写单变量数据格式**：
```
| 变量个数 (1B) | 地址 (4B) | 类型 (1B) | 数据 (NB) |
|      0x01     |           |           |           |
```

**批量写数据格式**：
```
| 变量个数 (1B) | 地址1 (4B) | 类型1 (1B) | 数据1 (NB) | 地址2 (4B) | ... |
```

### 2.3 数据类型编码

| 值 | 类型 | 字节数 | 说明 |
|----|------|--------|------|
| 0x01 | uint8 | 1 | 无符号8位整数 |
| 0x02 | int8 | 1 | 有符号8位整数 |
| 0x03 | uint16 | 2 | 无符号16位整数 |
| 0x04 | int16 | 2 | 有符号16位整数 |
| 0x05 | uint32 | 4 | 无符号32位整数 |
| 0x06 | int32 | 4 | 有符号32位整数 |
| 0x07 | float | 4 | 单精度浮点数 |

### 2.4 应答机制

| 命令 | 值 | 用途 | 数据格式 |
|------|-----|------|----------|
| ACK | 0xA0 | 操作成功 | 空 |
| NACK | 0xA1 | 操作失败 | 错误码 (1B) |

**错误码定义**：

| 值 | 含义 |
|----|------|
| 0x01 | 地址无效 |
| 0x02 | 类型无效 |
| 0x03 | 变量表满 |
| 0x04 | 校验错误 |

**应答规则**：
- 写命令成功返回ACK，失败返回NACK+错误码
- 读单次命令返回数据帧作为应答
- 周期读命令首次下发后，下位机持续周期发送数据帧，直到收到停止命令(0x1F)

### 2.5 通信流程示例

#### 单次读取

```
上位机 -> 下位机: [AA 55] [08] [01] [10] [地址4B] [CRC]
下位机 -> 上位机: [AA 55] [0B] [01] [地址4B] [数据4B] [CRC]
```

#### 周期读取

```
上位机 -> 下位机: [AA 55] [0C] [01] [13] [地址1 4B] [地址2 4B] [CRC]
下位机 -> 上位机: [AA 55] [0F] [01] [地址1 4B] [数据1 4B] [地址2 4B] [数据2 4B] [CRC]
下位机 -> 上位机: ... (每10ms发送一次)
上位机 -> 下位机: [AA 55] [04] [01] [1F] [CRC]  (停止)
```

#### 写入变量

```
上位机 -> 下位机: [AA 55] [0B] [01] [20] [01] [地址4B] [类型1B] [数据4B] [CRC]
下位机 -> 上位机: [AA 55] [04] [01] [A0] [CRC]  (ACK)
```

## 3. 下位机固件

### 3.1 目录结构

```
firmware/
├── sparam.c           # 核心实现
├── sparam.h           # 公开接口
├── sparam_hal.h       # HAL抽象层接口定义
├── sparam_port.h      # 配置文件
└── stm32/
    └── sparam_stm32.c # STM32参考实现
```

### 3.2 配置宏

在 `sparam_port.h` 中配置：

```c
#define SPARAM_DEVICE_ID        1                   // 设备ID
#define SPARAM_VARS_MAX_SIZE    32                  // 同时周期性监测的最大变量数
#define SPARAM_RX_BUF_SIZE      256                 // 接收缓冲区大小
#define SPARAM_TX_BUF_SIZE      512                 // 发送缓冲区大小
```

### 3.3 HAL接口

用户需实现以下接口（定义在 `sparam_hal.h`）：

```c
/**
 * @brief 通过串口DMA发送数据
 * @param data 数据指针
 * @param len 数据长度
 */
void sparam_uart_send(uint8_t *data, uint16_t len);
```

### 3.4 用户API

```c
/**
 * @brief 初始化sparam模块
 * @param device_id 设备ID
 */
void sparam_init(uint8_t device_id);

/**
 * @brief DMA接收完成回调，在DMA接收中断中调用
 * @param data 接收到的数据
 * @param len 数据长度
 */
void sparam_on_rx_done(uint8_t *data, uint16_t len);

/**
 * @brief 定时器中断回调，建议在1ms定时器中断中调用
 */
void sparam_on_timer_tick(void);
```

### 3.5 典型集成示例

```c
// main.c

#include "sparam.h"

// DMA接收缓冲区（乒乓缓冲）
uint8_t rx_buf[2][SPARAM_RX_BUF_SIZE];
volatile uint8_t rx_buf_idx = 0;

// 串口DMA发送实现
void sparam_uart_send(uint8_t *data, uint16_t len)
{
    HAL_UART_Transmit_DMA(&huart1, data, len);
}

// 串口接收完成回调
void HAL_UART_RxCpltCallback(UART_HandleTypeDef *huart)
{
    if (huart == &huart1) {
        uint8_t completed_buf = rx_buf_idx;
        rx_buf_idx = 1 - rx_buf_idx;
        HAL_UART_Receive_DMA(&huart1, rx_buf[rx_buf_idx], SPARAM_RX_BUF_SIZE);
        sparam_on_rx_done(rx_buf[completed_buf], SPARAM_RX_BUF_SIZE);
    }
}

// 定时器中断回调
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim)
{
    if (htim == &htim6) {
        sparam_on_timer_tick();
    }
}

int main(void)
{
    HAL_Init();
    SystemClock_Config();
    
    sparam_init(SPARAM_DEVICE_ID);
    
    HAL_UART_Receive_DMA(&huart1, rx_buf[0], SPARAM_RX_BUF_SIZE);
    HAL_TIM_Base_Start_IT(&htim6);
    
    while (1) {
        // 主循环
    }
}
```

### 3.6 内部实现要点

1. **变量表管理**
   - RAM中维护一个变量地址表，容量由 `SPARAM_VARS_MAX_SIZE` 限定
   - 周期读命令下发时，将地址加入表并记录采样速率
   - 停止命令清空变量表

2. **周期采样**
   - `sparam_on_timer_tick()` 在1ms定时器中调用
   - 内部维护计数器，按各变量配置的采样周期触发读取
   - 读取完成后通过DMA发送

3. **帧解析**
   - `sparam_on_rx_done()` 解析完整帧
   - 校验CRC后执行对应命令
   - 写操作返回ACK/NACK

## 4. 上位机工具

### 4.1 目录结构

```
host/
├── sparam/                 # Python包
│   ├── __init__.py
│   ├── protocol.py         # 协议编解码
│   ├── elf_parser.py       # ELF/MAP文件解析
│   ├── serial_conn.py      # 串口通信
│   └── device.py           # 设备管理
├── cli.py                  # CLI入口
├── gui.py                  # GUI入口
└── requirements.txt
```

### 4.2 依赖

- Python >= 3.8
- pyserial：串口通信
- pyelftools：ELF文件解析
- PyQt5 或 PySide6：GUI（可选）

### 4.3 CLI功能

```bash
# 列出串口
sparam list-ports

# 加载ELF文件，列出全局变量
sparam parse-elf firmware.elf

# 连接设备
sparam connect COM3 --device-id 1

# 读取变量
sparam read --var motor_speed --var motor_current

# 周期读取
sparam monitor --var motor_speed --rate 10 --output data.csv

# 写入变量
sparam write --var kp --value 1.5 --type float

# 停止监控
sparam stop

# 心跳检测
sparam ping
```

### 4.4 GUI功能

1. **连接管理**
   - 串口选择
   - 设备ID配置
   - 连接状态显示

2. **变量浏览器**
   - 加载ELF/MAP文件
   - 显示全局变量列表（名称、地址、类型、大小）
   - 搜索/过滤功能

3. **监测面板**
   - 选择变量加入监测列表
   - 配置采样速率
   - 实时数值显示
   - 实时曲线绘制

4. **参数标定**
   - 变量写入
   - 批量写入
   - 预设值管理
