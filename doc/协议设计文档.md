# sparam 协议设计文档

## 1. 协议目标与原则

sparam 协议用于上位机与下位机之间的参数读写与监测数据传输，目标是传输无关、结构简洁、易于 MCU 侧实现。

- 传输无关：协议与串口、Socket 等介质解耦
- 小端序统一：所有多字节字段均为小端序
- 可扩展：命令按高四位功能码、低四位参数拆分
- 可诊断：统一 ACK/NACK 与错误码机制

## 2. 帧格式

所有多字节字段均为小端序。

```
| 帧头 (2B) | 长度 (1B) | 设备ID (1B) | 命令 (1B) | 数据 (N B) | CRC16 (2B) |
|  AA  55   |           |             |           |            |   MODBUS   |
```

| 字段 | 大小 | 说明 |
|------|------|------|
| 帧头 | 2B | 固定 `0xAA 0x55` |
| 长度 | 1B | 命令 + 数据 + CRC 的总长度（不含帧头和长度字段） |
| 设备ID | 1B | 目标设备地址，0-255 |
| 命令 | 1B | 操作码 |
| 数据 | NB | 命令相关数据 |
| CRC16 | 2B | CRC16-MODBUS 校验，从设备ID开始计算 |

## 3. 命令编码

命令字节分为高4位（功能码）和低4位（参数）：

| 功能码 | 含义 |
|--------|------|
| 0x0x | 系统命令 |
| 0x1x | 读操作 |
| 0x2x | 写操作 |
| 0xAx | 应答 |

### 3.1 系统命令 (0x0x)

| 命令 | 值 | 用途 | 数据格式 |
|------|-----|------|----------|
| 心跳 | 0x00 | 检测设备是否在线 | 空 |
| 查询信息 | 0x01 | 查询设备信息/协议版本 | 空 |

查询信息返回：

```
| 协议版本 (1B) | 设备名称长度 (1B) | 设备名称 (NB) |
```

### 3.2 读操作 (0x1x)

低4位表示采样速率：

| 低4位 | 采样间隔 | 说明 |
|-------|----------|------|
| 0x0 | 单次 | 不启动周期，立即返回一次 |
| 0x1 | 1ms | 周期性上传 |
| 0x2 | 5ms | 周期性上传 |
| 0x3 | 10ms | 周期性上传 |
| 0x4 | 20ms | 周期性上传 |
| 0x5 | 50ms | 周期性上传 |
| 0x6 | 100ms | 周期性上传 |
| 0x7 | 200ms | 周期性上传 |
| 0x8 | 500ms | 周期性上传 |

| 命令 | 值 | 用途 | 数据格式 |
|------|-----|------|----------|
| 读单次 | 0x10 | 读取一次立即返回 | 请求数据见下表 |
| 周期读 | 0x11-0x18 | 按指定速率周期上传 | 请求数据见下表 |
| 停止采样 | 0x1F | 停止所有周期性上传 | 空 |

读请求数据格式：

```
| 地址1 (4B) | 地址2 (4B) | ... | 地址N (4B) |
```

读返回数据格式：

```
| 地址1 (4B) | 数据1 (NB) | 地址2 (4B) | 数据2 (NB) | ... |
```

### 3.3 写操作 (0x2x)

| 命令 | 值 | 用途 | 数据格式 |
|------|-----|------|----------|
| 写单变量 | 0x20 | 写入一个变量 | 见下表 |
| 批量写 | 0x21 | 写入多个变量 | 见下表 |

写单变量数据格式：

```
| 变量个数 (1B) | 地址 (4B) | 类型 (1B) | 数据 (NB) |
|      0x01     |           |           |           |
```

批量写数据格式：

```
| 变量个数 (1B) | 地址1 (4B) | 类型1 (1B) | 数据1 (NB) | 地址2 (4B) | ... |
```

## 4. 数据类型编码

| 值 | 类型 | 字节数 | 说明 |
|----|------|--------|------|
| 0x01 | uint8 | 1 | 无符号8位整数 |
| 0x02 | int8 | 1 | 有符号8位整数 |
| 0x03 | uint16 | 2 | 无符号16位整数 |
| 0x04 | int16 | 2 | 有符号16位整数 |
| 0x05 | uint32 | 4 | 无符号32位整数 |
| 0x06 | int32 | 4 | 有符号32位整数 |
| 0x07 | float | 4 | 单精度浮点数 |

实现约定（host 侧）：

- Host 代码中数据类型、CLI 类型字符串与 C 类型别名通过统一注册表维护。
- `DataType.size` 与 `DataType.format_char` 由注册表派生，避免重复硬编码。

## 5. 应答机制

| 命令 | 值 | 用途 | 数据格式 |
|------|-----|------|----------|
| ACK | 0xA0 | 操作成功 | 空 |
| NACK | 0xA1 | 操作失败 | 错误码 (1B) |

错误码定义：

| 值 | 含义 |
|----|------|
| 0x01 | 地址无效 |
| 0x02 | 类型无效 |
| 0x03 | 变量表满 |
| 0x04 | 校验错误 |

应答规则：

- 写命令成功返回 ACK，失败返回 NACK+错误码
- 读单次命令返回数据帧作为应答
- 周期读命令首次下发后，下位机持续周期发送数据帧，直到收到停止命令 (0x1F)

实现约定（host 侧）：

- 读命令与采样率映射由统一注册表维护（含单次与周期读）。
- Host 提供命令分类辅助（读命令判定、周期读判定），用于设备层收包与响应过滤。

## 6. 通信流程示例

### 6.1 单次读取

```
上位机 -> 下位机: [AA 55] [08] [01] [10] [地址4B] [CRC]
下位机 -> 上位机: [AA 55] [0B] [01] [地址4B] [数据4B] [CRC]
```

### 6.2 周期读取

```
上位机 -> 下位机: [AA 55] [0C] [01] [13] [地址1 4B] [地址2 4B] [CRC]
下位机 -> 上位机: [AA 55] [0F] [01] [地址1 4B] [数据1 4B] [地址2 4B] [数据2 4B] [CRC]
下位机 -> 上位机: ... (每10ms发送一次)
上位机 -> 下位机: [AA 55] [04] [01] [1F] [CRC]  (停止)
```

### 6.3 写入变量

```
上位机 -> 下位机: [AA 55] [0B] [01] [20] [01] [地址4B] [类型1B] [数据4B] [CRC]
下位机 -> 上位机: [AA 55] [04] [01] [A0] [CRC]  (ACK)
```

## 8. 回归与边界测试（Host）

已覆盖以下关键边界：

- CRC 校验失败帧应被拒绝。
- 非法帧头应被拒绝。
- 不完整帧应被拒绝。
- 空载荷基础帧应可完成编码/解码 roundtrip。
